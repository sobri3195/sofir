feat: Add restore default CPTs feature to recover missing menus

## Problem
Users reported that CPT menus (listing, profile, article, event, appointment) 
disappeared from WordPress admin sidebar after they were accidentally deleted 
through the SOFIR admin interface.

## Solution
Implemented a "Restore Default CPTs" feature that allows users to recover 
all 5 default custom post types and their taxonomies with a single click.

## Changes

### CptManager (includes/sofir-cpt-manager.php)
- Added `restore_default_post_types()` method
- Added `restore_default_taxonomies()` method
- Both methods restore seed data from `get_seed_post_types()` and `get_seed_taxonomies()`
- Fire action hooks for extensibility

### ContentPanel (includes/class-admin-content-panel.php)
- Added `handle_restore_default_cpts()` handler
- Added "ðŸ”„ Restore Default CPTs" button in UI
- Added confirmation dialog before restoration
- Added informative description and success message
- Registered admin_post action hook
